## 架構 cheat sheet

```
~/.skills-manager/                                  ← vault (source of truth)
├── skills-manager.db                              ← SQLite, schema v11
└── skills/
    ├── pack-routers/                               ← 8 auto-generated routers
    │   ├── marketing/SKILL.md                      ← L1 (frontmatter: description + when_to_use; body: L2 table)
    │   └── ...
    ├── sm-overview/SKILL.md                        ← 8 sm-* skills
    └── (189 actual skills, grouped by pack)

~/.claude/skills/                                   ← Claude Code 讀嘅位
├── (hybrid mode, 建議) pack-*, sm-*, essential skills only (~20 entries, ~900 tokens)
├── (full mode)   everything (~127 entries, ~7000 tokens)
└── (router_only) 只 pack-* routers (~8 entries, ~350 tokens)
```

**三層**：
- **L1** = pack router frontmatter：`description` (80-120 chars) + `when_to_use` (200-400 chars)。合併 cap 1536 chars。
- **L2** = `description_router` per skill，render 去 router body 個 table，配 "→ Pick for…" 標籤。
- **L3** = 真正 SKILL.md，Claude 覺得 match 嗰陣先 load。

**Key files**：
- `crates/skills-manager-core/src/skill_store.rs` — DB layer
- `crates/skills-manager-core/src/pack_seeder.rs` — `ensure_sm_pack()` idempotent seeder
- `crates/skills-manager-core/src/builtin_skills.rs` — 9 `include_str!` skills
- `crates/skills-manager-core/src/router_render.rs` — 輸出 router SKILL.md
- `crates/skills-manager-core/src/sync_engine/mod.rs` — `reconcile_agent_dir`
- `crates/skills-manager-core/src/migrations.rs` — DB schema + drift error
- `src-tauri/src/commands/` — Tauri IPC
- `src/lib/tauri.ts` — frontend IPC wrappers
- `src/views/PacksView.tsx`, `src/views/MySkills.tsx` — 主要 edit UI
- `src/components/Sidebar.tsx`, `src/components/RouterEditor.tsx`, `src/components/SkillDetailPanel.tsx`

## 實用 commands

```bash
# Coverage 檢查
sqlite3 ~/.skills-manager/skills-manager.db <<'EOF'
SELECT p.name,
  CASE WHEN p.router_description='' OR p.router_description IS NULL THEN 'no-L1' ELSE 'L1' END,
  CASE WHEN p.router_when_to_use='' OR p.router_when_to_use IS NULL THEN 'no-wh' ELSE 'when' END,
  (SELECT COUNT(*) FROM pack_skills ps JOIN skills s ON s.id=ps.skill_id
   WHERE ps.pack_id=p.id AND s.description_router IS NOT NULL AND s.description_router!='')
    || '/' || (SELECT COUNT(*) FROM pack_skills WHERE pack_id=p.id)
FROM packs p ORDER BY p.name;
EOF

# Materialized vs routed
ls ~/.claude/skills/
sm current                                       # active scenario + disclosure mode
sm pack list-routers                             # 邊個 pack 有 L1

# Re-sync
sm sync

# Build + run
pnpm install
cargo tauri dev                                  # GUI
cargo build -p skills-manager-cli --release      # CLI
cp target/release/sm ~/.local/bin/sm             # 裝去 PATH
cargo test                                       # Rust tests
pnpm run lint
```
