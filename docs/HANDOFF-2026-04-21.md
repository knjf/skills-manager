## Architecture cheat sheet (for context)

```
~/.skills-manager/                                  ← vault (source of truth)
├── skills-manager.db                              ← SQLite, schema v11
└── skills/
    ├── pack-routers/                               ← 7 auto-generated routers
    │   ├── marketing/SKILL.md                      ← L1 (frontmatter: description + when_to_use; body: L2 table)
    │   ├── research/SKILL.md
    │   └── ...
    ├── sm-overview/SKILL.md                        ← 8 sm-* skills teaching Claude how to use Skills Manager
    ├── sm-scenarios/SKILL.md
    └── (189 actual skills, grouped by pack)

~/.claude/skills/                                   ← what Claude Code actually loads
├── (hybrid mode) pack-*, sm-*, essential-pack skills only (~20 entries, ~900 tokens)
├── (full mode)   everything (~127 entries, ~7000 tokens)
└── (router_only) just pack-* routers (~8 entries, ~350 tokens)
```

**Three tiers**:
- **L1** = pack router's `description` (80-120 chars) + `when_to_use` (200-400 chars) in frontmatter. Combined cap 1536 chars.
- **L2** = `description_router` per skill in the pack, rendered into router body as a table with "→ Pick for…" markers.
- **L3** = the actual SKILL.md, loaded by Claude when the user's task matches.

**Key files**:
- `crates/skills-manager-core/src/skill_store.rs` — DB layer
- `crates/skills-manager-core/src/pack_seeder.rs` — `ensure_sm_pack()` idempotent seeder
- `crates/skills-manager-core/src/builtin_skills.rs` — 9 `include_str!` skills
- `crates/skills-manager-core/src/router_render.rs` — emits router SKILL.md
- `crates/skills-manager-core/src/sync_engine/mod.rs` — `reconcile_agent_dir` (respects disclosure mode)
- `src-tauri/src/commands/` — Tauri IPC handlers
- `src/lib/tauri.ts` — frontend IPC wrappers + types
- `src/views/PacksView.tsx`, `src/views/MySkills.tsx` — main edit UIs
- `src/components/Sidebar.tsx`, `src/components/RouterEditor.tsx`, `src/components/SkillDetailPanel.tsx`
