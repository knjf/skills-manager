# Phase 2: CLI Implementation Plan

## Tasks

### Task 1: Create crate skeleton
- Create `crates/skills-manager-cli/Cargo.toml` with clap + core deps
- Add to workspace members in root `Cargo.toml`
- Create `src/main.rs` with clap derive structs for all commands
- Create `src/commands.rs` stub
- Verify: `cargo check -p skills-manager-cli`

### Task 2: Implement `sm list` and `sm current`
- `list`: Get all scenarios, get active ID, format with arrow marker
- `current`: Get active scenario, show name + effective skill count
- Both read-only, simplest starting point

### Task 3: Implement `sm skills [name]`
- Optional scenario name argument, defaults to active
- List effective skills for the scenario
- Case-insensitive scenario name lookup (shared helper)

### Task 4: Implement `sm diff <a> <b>`
- Find both scenarios by name
- Get effective skills for each
- Show skills only in A, then only in B

### Task 5: Implement `sm switch <name>`
- Find scenario by name
- Unsync old scenario: for each enabled adapter, remove SM-managed entries
- Set new active scenario in DB
- Sync new scenario: for each effective skill, sync to each enabled adapter
- Print progress per agent

### Task 6: Implement `sm packs [name]`
- List packs attached to a scenario
- Optional scenario name, defaults to active

### Task 7: Implement `sm pack add/remove`
- Add or remove a pack from a scenario by name
- Look up pack and scenario by name

### Task 8: Testing + verification
- Run all `cargo test` in workspace
- Build release binary
- Test against real DB

### Task 9: Simplify review
- Check for duplication, code quality
- Ensure consistent error messages

## Commit Strategy

One commit per task (or logical grouping of 2-3 tasks if small).
